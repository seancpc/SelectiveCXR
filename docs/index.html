SelectiveCXR

具統計保證的選擇性胸腔 X 光判讀系統

Python · PyTorch · vLLM · transformers · Conformal Prediction · Gradio · RTX 4090

一個「會說我不確定」的 AI 判讀系統 —— 有把握就自動判讀,沒把握就誠實標記、交給人類專家。 以胸腔 X 光(CXR)驗證,但框架領域無關:同一套邏輯可用於缺陷檢測、金融風控等任何 「錯誤成本高 + 需要可證明信任」的場景。

設計概念

這個專案不拼判讀準確率(那是紅海),而是拼 可信賴的自動化 —— 讓 AI 知道自己何時不可靠,把不確定的交給人。三個核心:

  • 選擇性棄答:有把握自動判讀、沒把握主動棄答轉人工,而不是對每張片子都硬給答案。
  • 多源不確定性 + 統計保證:用三個跨 backbone 模型的判斷分歧量化不確定性,再用 conformal prediction 給出「自動判讀部分錯誤率 ≤ α」的 distribution-free 保證。
  • 範式無關:前端判讀模型可替換(VLM / CNN / 混合),中間的不確定性與決策層完全不動 —— 這是「醫療 → 跨領域」可遷移的技術支點。

專案簡介

像一個好的實習醫師:大部分片子能獨立判讀,但知道自己什麼時候該去問主治醫師。現在的醫療 AI 最危險的不是「看錯」,而是「看錯了還很自信」—— 對每張片子都給斬釘截鐵的答案。SelectiveCXR 要做的是有「自知之明」的 AI。

  • 任務:胸腔 X 光的 14 種 CheXpert 病徵 multi-label 判定 + 三檔分流決策。
  • 驗證規模:MIMIC-CXR,calibration 1000 + test 671 張,14 病徵,α = 0.15 操作點。

系統架構

三個「出身不同」的視覺語言模型各自判讀同一張片子,用它們之間的判斷分歧當不確定性訊號,經 conformal 校準後做三檔分流。

              胸腔 X 光
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
  Qwen3-VL    MedGemma    Llama-3.2-V      ← 三個跨 backbone 的 VLM
   (通用)      (醫療)       (通用)            錯誤不相關,分歧才有意義
     └────────────┼────────────┘
                  ▼
       跨模型 disagreement(判斷分歧)        ← 不確定性訊號
                  ▼
        Conformal Prediction 校準           ← distribution-free 統計保證
                  ▼
            三檔分流決策
     ┌────────────┼────────────┐
     ▼            ▼            ▼
   高度自動    部分標記    整案轉人工
  • 跨 backbone ensemble:刻意選三個不同來源的模型(Qwen / Gemma / Llama)。架構多樣性 → 錯誤不相關 → 它們「吵架」時才是真的難 case。
  • disagreement 當訊號:三模型對同一病徵越分歧,越可能棄答。
  • Conformal Prediction:用統計方法決定棄答門檻,給白紙黑字的錯誤率保證,而非工程師手調。
  • 三檔分流:高度自動 / 部分標記待確認 / 整案轉人工,並標出系統「天生判不準、固定交人工」的能力邊界。

使用技術

類別技術堆疊
模型(自架)Qwen3-VL-8B、MedGemma-1.5-4b、Llama-3.2-11B-Vision(4-bit 量化,三模型共 ~17 GB / 24 GB)
推論引擎vLLM(continuous batching,~28× 提速)+ transformers(混合引擎,跑 vLLM 不支援的 Mllama)
不確定性與決策跨模型 disagreement、Conformal Prediction(Mondrian / per-label)、三檔分流
對比實驗TorchXRayVision DenseNet(CNN baseline)、per-label AUC(不依賴 threshold)
前端Gradio(白話化 demo,病徵中文、合併視覺化)
資料MIMIC-CXR(CheXpert 14-label);demo 展示用公開 NIH ChestX-ray14
硬體 / 環境單張 RTX 4090(WSL2)、Python

成效亮點

  • 選擇性判讀核心驗證:棄答最不確定的樣本後,自動判讀部分的準確率明顯優於全答(全答 ~0.80)。
  • 能力邊界 = 2 / 14:系統標出 disagreement 救不回的兩個高陽性率病徵(Cardiomegaly、Lung Opacity)固定交人工,其餘做選擇性判讀。
  • CNN vs VLM(per-label AUC,不依賴 threshold):CNN 0.626 > VLM ensemble 0.585 —— VLM 判斷品質略遜專用 CNN(誠實,不誇大);但兩者逐病徵互有勝負、錯誤不相關(CNN 強密度型病灶、VLM 強型態型),這正是「CNN + VLM 混合」的理論基礎。
  • 多標籤聯合棄答(結構性發現):在嚴格 α=0.05 下,每張片子棄答數高度集中、「整張零棄答」幾乎不發生 —— 揭示多標籤 selective prediction 的結構性挑戰。展示採 α=0.15 操作點,使三檔都能呈現;α=0.05 的真實分布如實記錄,展示操作點 ≠ 隱藏真相。

開發歷程(誠實的工程判斷)

開發過程不是「一路成功」,而是一連串驗證 → 發現問題 → 修正:

  • ✅ disagreement 訊號有效:選擇性判讀後,自動判讀部分的準確率顯著高於全答。
  • ❌ 驗證後放棄一個無效訊號:曾假設「模型對病灶定位(grounding)的分歧」可當不確定性訊號,但 AUC 嚴格驗證後發現無區辨力(VLM 判斷可靠、定位不可靠),選擇移除而非硬塞無效訊號。
  • 🔧 看穿真正的約束:第三模型候選一再卡在推論框架的版本相容性,而非模型本身。意識到「約束是 vLLM 而非模型」後,改用 transformers 跑 Llama-3.2-Vision,回到理想的跨 backbone 設計。
  • 📊 推翻自己:初版 CNN vs VLM 對比一度顯示「VLM 碾壓 CNN」,但 CNN 準確率 0.188 這個荒謬數字觸發警覺 —— 發現是 threshold 假象,改用不依賴 threshold 的 per-label AUC 重做,得到誠實結論。

未來可擴展性

  • CNN + VLM 混合:對比實驗已證明兩者錯誤不相關、逐病徵互有勝負 —— 把 CNN 接進同一個 selective + conformal 框架,可截長補短(框架範式無關,決策層不需改動)。
  • 跨領域遷移:同時滿足「錯誤成本高 / 有人工 fallback / 需可證明信任」的場景皆適用 —— 缺陷檢測、金融風控、保險核保等;把胸腔 X 光換成其他影像,只需更換前端判讀模型。
